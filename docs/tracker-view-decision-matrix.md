# Decision Matrix: Tree collapse + editor access

Контекст: вкладка «Дерево артефактов» (`/?project=<id>&tab=artifacts`) в
`tracker-view/tracker-view.mjs`. Пользователь дал две гипотезы:

1. **collapse**: «добавил бы раскрытие-сворачивание если там есть что сворачивать»
2. **editor access**: «элементы делать доступными для редактора (?)» — гипотеза.

Замер текущего состояния (granite, project=17, 76 артефактов):
- `episode` — всегда `open` (1 шт.)
- `anode-det` (узлы с детьми) — БЕЗ `open`, все свёрнуты (3 шт.)
- title узла → `/?artifact=<id>` (просмотр); кнопка ✎ только на странице просмотра
- 0 прямых ссылок в `/artifact/N/edit` из дерева; 0 иконок-карандашей

## Collapse-стратегии

- **c1**: все узлы `open` по умолчанию + можно свернуть вручную.
- **c2**: запоминать open-состояние в localStorage (по id артефакта).
- **c3**: кнопки «развернуть всё / свернуть всё» сверху дерева.
- **c4**: текущее поведение — episode open, узлы-родители свёрнуты.

## Editor-access-стратегии

- **e1**: как есть — title → просмотр, оттуда кнопка ✎.
- **e2**: иконка-карандаш ✎ прямо в каждом узле рядом с title → `/artifact/N/edit`.
- **e3**: двойной клик → edit, одинарный → просмотр.
- **e4**: title → edit напрямую (без промежуточной страницы просмотра).

## Критерии (столбцы)

- **Сложность↓** (1=просто, 5=сложно) — минимизировать.
- **Полезность↑** (1-5) — максимизировать.
- **Риск↓** (1=низкий, 5=высокий) сломать существующее — минимизировать.
- **UX-соглас.↑** (1-5) с GitHub-dark / нативный `<details>` стиль — максимизировать.

Итог = `Полезность + UX-соглас − Сложность − Риск` (максимум при +8).

## Матрица с оценками

| Вариант | Слож↓ | Полез↑ | Риск↓ | UX↑ | ИТОГ |
|---|---|---|---|---|---|
| **A**: c4 + e1 (как есть, ничего не менять) | 1 | 1 | 1 | 3 | **+2** |
| **B**: c3 + e2 — кнопки развернуть/свернуть всё + ✎-карандаш в узле | 2 | 5 | 1 | 5 | **+7** |
| **C**: c1 + e2 — все узлы open + ✎-карандаш в узле | 2 | 4 | 2 | 3 | **+3** |
| **D**: c2 + e3 — localStorage + dblclick→edit/single→view | 4 | 4 | 3 | 2 | **−1** |
| **E**: c3 + e4 — кнопки разв./свер. + title → edit напрямую | 2 | 4 | 3 | 2 | **+1** |
| **F**: c1 + c3 + e2 — все open + кнопки + ✎ (максимальный) | 3 | 5 | 2 | 4 | **+4** |

## Решение и обоснование

**Выбран вариант B: c3 (кнопки «развернуть всё / свернуть всё») + e2 (✎-карандаш в узле).**

Почему:
- **Полезность 5/5** — закрывает обе гипотезы пользователя напрямую: управляемое
  сворачивание (видеть дерево целиком или свернуть лишнее) + мгновенный доступ к
  редактору из любого узла без промежуточной страницы.
- **Риск 1/5** — чисто аддитивные правки: добавляем тулбар и одну ссылку-иконку в
  `renderNode`. Никаких изменений в роутинге, БД, partial-логике, save-хендлере.
  Обратная совместимость 100%: существующие `/?artifact=N` и кнопка ✎ на странице
  просмотра остаются.
- **UX-согласие 5/5** — карандаш ✎ уже используется на странице просмотра (`✎ Редактировать`),
  тулбар в стиле `.filter-bar`/`.chip` (существующие компоненты), нативный `<details>`
  сохраняется, refresh-логика `partial=2` не ломается (toolbar вне `.episodes`).
- **Сложность 2/5** — небольшой JS для toggle-all + правка шаблона узла.

Почему **не** C/F (всё open по умолчанию): дерево granite = 76 артефактов на 3 уровнях;
полностью развёрнутое дерево = длинная простыня, теряется обзор. Кнопка «свернуть всё»
дает тот же контроль без потери дефолтной компактности (узлы-родители остаются свёрнутыми,
как сейчас — пользователь разворачивает целенаправленно).

Почему **не** D (localStorage + dblclick): localStorage хрупок (сбрасывается при
auto-refresh `partial=2`, который переписывает `.episodes`), а dblclick — скрытый
hard-to-discover affordance без визуальной подсказки, расходится с GitHub-dark UX.

Почему **не** E (title → edit напрямую): ломает обратную совместимость — все пользователи,
привыкшие к «клик по title = чтение», попадут в форму. Просмотр должен остаться дефолтом
(как в GitHub wiki). Карандаш ✎ — явное отдельное действие.

## Что реализуется (детально)

1. **Тулбар дерева** (новый `.tree-toolbar` над `.episodes`): счётчик + 2 кнопки
   `.chip` — «▸ Развернуть всё» / «▾ Свернуть всё». JS переключает `open` у всех
   `details` внутри `.episodes` (episode + anode-det).
2. **✎-карандаш в узле** (новый `<a class="aedit">`): появляется у КАЖОГО узла с
   существующим `.md`-файлом (проверка `resolveArtifactFile` была бы дорогой на
   сервере → ставим карандаш у всех узлов; редирект `/artifact/N/edit` сам покажет
   «файл будет создан», как сейчас в редакторе). CSS: маленькая иконка справа от
   status, ховер → подсветка, `title="Редактировать .md"`.
3. **Сохранение состояния при auto-refresh**: расширяем существующий механизм
   `openKeys` (строки 696-704) — он уже работает по textContent summary; после
   toggle-all пользователем refresh не сбросит ручное состояние.

## Что НЕ реализуется (и почему)

- **localStorage** (c2): см. выше — сбрасывается при partial-рефреше, низкий ROI.
- **Inline-редактирование** (textarea прямо в узле): огромная сложность (markdown-
  editor в дереве, save без перезагрузки, race с partial-refresh), риск 5/5. Карандаш
  → полноценная страница редактирования покрывает потребность с риском 1/5.
- **dblclick** (e3): скрытый affordance, ломает mobile/touch.
